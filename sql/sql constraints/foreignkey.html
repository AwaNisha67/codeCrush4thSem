SQL FOREIGN KEY constraint

Foreign Key is a column/field that refers to the primary key/unique key of another table. So it demonstrates the relationship between tables and acts as a cross reference among them.

  • A Foreign Key Constraint establishes a link between the data in two tables.
  • It consists of a column or combination of columns in one table that holds values corresponding to the primary key values in another table.
  • This linkage creates a relationship between the two tables often referred to as a parent-child relationship.
  • The table with the primary key is termed the Parent Table. The table with the foreign key is termed the Child Table.
  • This relationship is crucial for maintaining Referential Integrity in the database.

CREATE FOREIGN KEY:-

• Using CREATE TABLE statement:-

CREATE TABLE tableName( colName1 dataType, colsName2> datatype .... FOREIGN KEY(colname)REFERENCESotherTableName(colName));

• Using ALTER TABLE statement:-

ALTER TABLE tableName
ADD FOREIGN KEY,(colname)REFERENCES otherTableName(colName);

DROPING A FOREIGN KEY:-

If you want to delete the foreign key from the column of the table, then you have to use the following ALTER syntax in SQL:

Syntax:-

ALTER TABLE tableName
DROP FOREIGN KEY constraint_name;
