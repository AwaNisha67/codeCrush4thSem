SQL PRIMARY KEY constraint

The Primary Key Constraint uniquely identifies each record in a database table

  • A primary key column cannot contain NULL values.
  • Each table should have a primary key
  • Each table can only have one primary key.
  • When a primary key is applied to a single attribute, it is called a Simple Key.
  • When a primary key is applied to two or more than two attributes, it is called a Composite Key.
  • Composite keys are useful when no single attribute can uniquely identify a record ,but a combination of attributes can.

CREATE PRIMARY KEY:-

• Using CREATE TABLE statement:-

CREATE TABLE tableName( colName1 dataType PRIMARY, colsName2> datatype .... );

Example:-

CREATE TABLE student(rollNo INT PRIMARY,
 firstName VARCHAR(10),lastName VARCHAR(10),age DATE );

• Using ALTER TABLE statement:-

ALTER TABLE tableName
ADD PRIMARY KEY, (columnName);

Example:-

ALTER TABLE student ADD PRIMARY KEY rollNo;

DROP PRIMARY KEY:-

If you want to delete the Primary key from the column of the table, then you have to use the following ALTER syntax in SQL:

Syntax:-

ALTER TABLE tableName
DROP PRIMARY KEY;

Example:-

ALTER TABLE student DROP PRIMARY KEY;
